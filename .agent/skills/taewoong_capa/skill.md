# 1. 15,000T 프레스 물리적 제약 조건 (CAPA Validation Rules)

- **상수 모듈화:** 모든 물리적 제약 조건 수치(최대 거리, 허용 중량 등)는 반드시 `config.js` 또는 `constants/` 폴더 내에 상수로 분리하여 사용자가 쉽게 숫자를 수정할 수 있도록 코드를 모듈화하라.
- **검증 1 (M.P - Die 거리 한계):** 사용자가 입력한 단조품의 외경(OD) 또는 높이가 `M.P(Moving Platen)와 Die 사이의 최대 거리`를 초과하는지 실시간으로 검증하라.
- **검증 2 (SHELL 형상 한계):** 형상 콤보박스에서 'SHELL'이 선택된 경우, 15,000T 프레스가 타각할 수 있는 최대 중량(Max Weight) 및 최대 길이/외경 한계값을 엄격히 적용하여 검증하라.
- **직관적 UX (경고 UI):** 검증을 통과하지 못할 경우, 폼 입력창에 붉은색 경고를 렌더링하라. 단순한 에러 메시지가 아닌 "왜 이 치수가 불가능한지 (예: M.P-Die 거리 초과, SHELL 최대 중량 초과)"를 명확한 툴팁이나 경고 텍스트로 알려주어야 한다.

# 2. 스마트 팩토리 UI/UX 및 워크플로우 규칙

- **디자인 무드:** 차분한 블루/그레이 톤을 메인 컬러로 사용하여 직관적이고 신뢰감을 주는 '스마트 팩토리 대시보드 및 제어판' 컨셉을 유지하라.
- **스마트 입력 폼:** NDT 규격(ASME, ASTM, EN 등), 열처리 조건(냉각 매질, Q/T 등), 형상(Ring, Shell 등)은 데이터 일관성을 위해 모두 '검색 기능이 포함된 콤보박스(Select Dropdown)' UI로 구현하라.
- **실무 워크플로우 & 리비전:** 시방서 문서는 [작성 중 -> 검토 요청 -> 승인 완료]의 상태(Status) 관리 로직을 포함해야 하며, 문서가 수정될 때마다 `Rev.1`, `Rev.2` 형식으로 리비전 이력이 자동 추적되도록 DB를 연동하라.
- **PDF 출력 디자인:** 출력 화면 상단에는 '주식회사 태웅(Taewoong)' 로고와 문서 번호(리비전 포함)를 박아넣고, 본문 내용은 현장 작업자가 한눈에 볼 수 있도록 깔끔한 표(Table) 형식의 정식 시방서 양식을 적용하라.

# 3. AI 공정 추론 연동 로직

- 검증을 통과한 [치수/중량/형상] 및 [소재/물성치] 데이터가 주어지면, AI 기능을 활용해 최적의 단조 온도 구간, 가열 유지 시간, 열처리 사이클을 추론하여 제안하는 프롬프트 체인 및 UI 영역을 구성하라.
- 선택된 국제 표준 규격(예: ASTM A336)의 범위를 벗어나는 입력이 발생하면 경고 메시지를 출력하도록 로직을 설계하라.
